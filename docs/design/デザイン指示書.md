## 1. プロジェクト概要

- **プロジェクト名：** DevForge
    
- **ビジョン：** 「エンジニアの現在地を可視化し、未来への歩みを整える」
    
- **目的：**
    
    - エンジニア個人：自身のスキル・経歴の整理と市場価値の把握、ポートフォリオ公開。
        
    - 管理・営業側：エンジニアのスキル状況の俯瞰、適切な案件アサインの判断材料。
        
- **ターゲット：** ITリテラシーの高いエンジニア、および管理職・営業職。
    

## 2. デザインコンセプト

**「Professional Workbench（プロフェッショナルの作業台）」**

AIが生成したような過度な装飾や無意味な余白を排し、データ密度と実用性を最優先する。エンジニアが日常的に触れる「開発ツール（IDEやダッシュボード）」のような、整然とした美しさとワクワク感を追求する。

## 3. ビジュアル・アイデンティティ

### 3.1 カラーシステム（Material Design 3 準拠）

- **Primary:** 深みのある青（信頼と誠実、先進性）。
    
- **Surface:** ライトモードをデフォルトとし、背景には純白ではなく、ごく薄いグレー（M3 Surface Container Low）を使用して眩しさを抑え、カードの白を際立たせる。
    
- **State:** エラー（赤）、警告（黄）、成功（緑）はM3の標準トーンを使用。
    
- **Contrast:** WCAG 2.2 Level AA（コントラスト比4.5:1以上）を全テキストで厳守。
    

### 3.2 タイポグラフィ

- **基本フォント:** `Noto Sans JP`, `Inter`（可読性の高いサンセリフ体）。
    
- **データフォント:** `JetBrains Mono`（数字、スキル名、バージョン表記等に使用。ツール感を演出する）。
    
- **密度:** 行間（leading）は1.6前後とし、ツールライクな情報密度を保ちつつ、読み飛ばしを防ぐ。
    

### 3.3 形状と質感

- **Radius（角丸）:** Shadcn UI / M3の標準（8px程度）を基本とする。硬すぎず、適度な柔らかさを残す。
    
- **Elevation:** 影による表現は最小限とし、境界線（1pxのボーダー）と色の面（Surface）で階層を表現する。
    

## 4. レイアウト・UX設計

### 4.1 ナビゲーション

- **Navigation Rail (Left):** 左側に固定の縦型メニュー。
    
    - スリムなアイコン表示をデフォルトとし、コンテンツ領域を最大化。
        
    - エンジニアが効率的に画面遷移できるよう、常に表示。
        

### 4.2 主要画面：ダッシュボード（U001）

- **構成:**
    
    - **サマリー:** 入力完了率プログレスバーによる「積み上げ実感」の提供。
        
    - **スキル・マトリックス・テーブル:**
        
        - `Tabs`コンポーネントでカテゴリー（言語、FW、ツール等）を切り替え。
            
        - `Table`形式で、スキル名、経験年数、習熟度（5段階バー）、最終利用日を羅列。
            
        - ホバー時に Primary Blue の薄いハイライトを入れ、操作性を高める。
            
- **インタラクション:**
    
    - ページ遷移やデータの保存時に、簡易的でスムーズなアニメーション（フェードやスライド）を入れ、モダンな操作感を演出。

## 6. 技術実装ガイドライン

- **Frontend:** Nuxt.js (Vue.js)
    
- **Styling:** Tailwind CSS + Shadcn UI
    
- **Component Library:** Shadcn UI（カスタマイズしてM3の挙動に寄せる）
    
- **Accessibility:**
    
    - WCAG 2.2 準拠。
        
    - キーボード操作（Tab移動）の完全対応。
        
    - aria-label の適切な設定。
